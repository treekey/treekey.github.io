Windows 套件管理工具 (Scoop, Chocolatey)

身為開發環境毀滅者，常常會因為某些原因，需要在Windows上安裝一些開發用的軟體。要自行去網路上尋找、下載，有時候還會找不到檔案，或是找到由可疑的非官方頁面所提供的安裝檔，下載回 來還需要執行安裝檔案，通過繁瑣的下一步、下一步、下一步，還得定義安裝路徑。這時候就特別羨慕Linux的apt-get的使用方式，因為提供了command-line的形式，可以很簡單的寫成腳本的形式，在腳本裡定義自己的需求列表，在新機建置環境時能一鍵完成，特別方便。

接下來會介紹兩個各有優缺點的Windows套件管理工具，分別是Chocolatey和Scoop，都需要先安裝，然後就能使用提供的command來搜尋、安裝、移除來管理其他套件了。筆者是在Windows 10作業系統上測試的，如果是使用Windows 7或是Windows Server 2008這些比較舊的作業系統的使用者，請先安裝Windows Management Framework 5.1以確保能執行Windows PowerShell。

Chocolatey

Chocolatey 安裝需要先以系統管理員權限打開PowerShell

如何賦予權限?
(在開始列搜尋Windows PowerShell，以右鍵點擊選擇以系統管理員身分執行)
如果沒有權限，則安裝會失敗並顯示錯誤

然後在PowerShell輸入下列指令來安裝

1Set-ExecutionPolicy Bypass -Scope Process -Force; 
2[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; 
3iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

安裝後先執行choco，如果安裝成功會顯示版號

1Chocolatey v0.10.15
2Please run 'choco -?' or 'choco <command> -?' for help menu.

根據提示執行chaco -?，可以進一步看到可使用的指令，以下列出簡單用法

 1# 安裝套件: choco install PACKAGE_NAME
 2# 以vscode為例
 3choco install vscode
 4
 5# 移除套件: choco uninstall PACKAGE_NAME
 6choco uninstall vscode
 7
 8# 檢查是否有套件可更新
 9choco outdated
10
11# 更新套件: choco upgrade PACKAGE_NAME
12choco upgrade vscode
13
14# 搜尋套件: choco search PACKAGE_NAME
15choco search vscode
16
17# 確認套件資訊，像是版本、發佈日期、介紹等等: choco info PACKAGE_NAME
18choco info vscode

透過choco install的套件一般都會放在C:\Program Files或C:\Program Files (x86)之中，但也有些工具軟體會被放在C:\Tools 。

Scoop

Scoop的特色是安裝不需要權限

首先一樣用PowerShell先允許執行來自遠端的指令碼

1Set-ExecutionPolicy RemoteSigned -scope CurrentUser

接著才能執行安裝指令

1iwr -useb get.scoop.sh | iex

安裝後可以直接輸入scoop來確認是否安裝成功，若成功會顯示可用的相關指令，基本上用法跟Chocolatey相去不遠

 1# 安裝套件: scoop install PACKAGE_NAME
 2# 以curl為例
 3scoop install curl
 4
 5# 移除套件: scoop uninstall PACKAGE_NAME
 6scoop uninstall curl
 7
 8# 檢查是否有套件可更新
 9scoop status
10
11# 更新套件: scoop update PACKAGE_NAME
12scoop update curl
13
14# 搜尋套件: scoop search PACKAGE_NAME
15scoop search curl
16
17# 確認套件資訊，像是版本、License
18scoop info curl

透過scoop install的套件一般都會放在C:\Users\<YOUR_USER_NAME>\scoop

結論

從安裝跟使用方式來看，上述兩種管理工具看起來差不多，真要比較的話，Chocolatey提供的GUI套件較多，而且因為是用管理員權限去管理套件，因此套件間的相依性都幫我們處理掉了，所以很適合一般沒有程式開發背景的使用者。而Scoop雖然提供的GUI套件相對來說沒那麼多，但是安裝的套件被限定在你的使用者帳戶中，且被放在統一的目錄裡，避免了路徑汙染，但有些依賴性較強的套件在安裝上就需要額外的配置動作才能完成安裝。因此對有點程式背景的我來說，首選會是Scoop，真的不行或是想懶了才會改用Chocolatey。
